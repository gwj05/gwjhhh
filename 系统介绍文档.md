## 系统概述

### 项目简介

**智慧农业管理系统**是一个基于Web的现代化农业管理平台，旨在通过信息化手段提升农场管理效率，实现农业生产的智能化、精细化管理。系统集成了农场管理、作物管理、环境监测、智能预警、负责人管理等多个核心功能模块，为农业从业者提供全方位的数字化管理解决方案。

### 系统定位

- **目标用户**：农场管理者、农业技术人员、农场负责人、普通农民
- **应用场景**：现代化农场、农业合作社、农业示范基地
- **核心价值**：提升管理效率、降低运营成本、实现精准农业

### 系统特点

- ✅ **多农场管理**：支持多农场统一管理，灵活切换
- ✅ **智能预警**：基于环境数据自动判断农场/作物状态
- ✅ **权限精细**：基于角色的权限控制（RBAC）
- ✅ **数据可视化**：环境趋势图表、健康度分析
- ✅ **实时监控**：实时环境数据采集与展示
- ✅ **操作追溯**：农事操作、采购入库、库存流水可追溯
- ✅ **农资全链路**：农资档案、采购下单、自动入库、库存预警
- ✅ **智能预警闭环**：设备与规则、异常记录、推送、统计与首页联动

## 核心功能模块

### 1. 用户认证与权限管理

#### 1.1 用户登录注册
- 用户注册（支持三种角色：管理员、运维人员、普通用户）
- 用户登录（JWT Token认证）
- 密码加密存储（bcrypt加密）
- 角色切换功能（管理员可切换角色）

#### 1.2 权限控制
- **角色体系**：
  - 超级管理员：系统最高权限，可管理所有农场
  - 农场管理员：管理指定农场的所有功能
  - 普通用户：查看和操作权限受限
- **路由保护**：基于Token的前端路由守卫
- **API权限**：后端中间件验证用户权限

### 2. 农场管理模块

#### 2.1 农场列表管理
- **查询筛选**：
  - 农场名称模糊搜索
  - 创建时间区间筛选
  - 按农场名称/创建时间排序
- **农场状态标签**：
  - 正常（绿色）：所有环境指标正常
  - 预警（黄色）：1项指标异常
  - 告警（红色）：2项及以上指标异常
- **悬浮卡片**：鼠标悬停显示实时温度、湿度、未处理预警数
- **批量操作**：批量导出Excel（农场名称、地址、负责人）
- **CRUD操作**：
  - 新增农场（名称、地址、经纬度、负责人）
  - 编辑农场信息
  - 删除农场（仅超级管理员）

#### 2.2 农场详情管理
- **多农场切换**：
  - 悬浮式农场选择面板
  - 支持搜索、最近访问、快速切换
  - 实时状态徽章显示
- **基础信息管理**：
  - 农场名称、地址、经纬度
  - 负责人信息（姓名、电话）
  - 创建时间
  - 支持编辑（除ID外所有字段）
- **关联数据管理**：
  - 关联作物管理（查询、添加、编辑、删除）
  - 关联设备管理（查询、绑定、编辑、解绑）
- **数据可视化**：
  - 24小时温度/湿度趋势图（折线图）
  - 7天土壤pH趋势图（柱状图）
  - 风速/降雨量统计（饼图）
  - 时间范围切换（24h/7天/30天）
- **智能分析**：
  - 作物健康度分析（环形进度条）
  - 健康度计算公式展示
  - 关键影响因素提示
- **智能建议**：
  - 灌溉建议（基于作物类型、生长周期、实时数据）
  - 施肥建议（精准推荐）
  - 建议内容可复制

### 3. 负责人管理模块

#### 3.1 负责人列表
- **查询筛选**：
  - 负责人姓名模糊搜索
  - 手机号精准搜索
  - 权限范围筛选（单个/多个农场管理）
  - 按绑定时间降序排序
- **列表展示**：
  - 姓名、手机号、角色
  - 权限范围（灰色标签）
  - 绑定时间
  - 主/副负责人标签（红色/蓝色）

#### 3.2 负责人绑定
- 下拉选择农场管理员
- 支持单个/批量绑定
- 绑定前检查（提示已绑定其他农场）
- 绑定记录日志

#### 3.3 权限配置
- **模块权限**：可查看的模块（作物/设备/预警/环境）
- **操作权限**：仅查看/可编辑/可删除
- 超级管理员可配置所有权限
- 农场主负责人仅能分配副负责人权限

#### 3.4 解除绑定
- 二次确认提示
- 仅超级管理员/主负责人可操作
- 解绑主负责人时，提示是否同步解绑副负责人
- 操作日志记录

#### 3.5 操作日志
- 记录绑定/解绑/改权限操作
- 支持按时间/操作类型筛选
- 导出Excel功能

### 4. 作物管理模块

#### 4.1 作物列表管理
- **多维度筛选**：
  - 作物名称模糊搜索
  - 作物类型下拉（果蔬/粮食/经济作物）
  - 种植农场下拉/模糊搜索
  - 种植状态下拉（生长中/成熟/已收割）
  - 种植时间区间筛选
- **智能状态标签**：
  - 正常（绿色渐变）：所有环境指标在适宜区间
  - 预警（黄色渐变）：1项指标超出适宜区间
  - 告警（红色渐变）：2项及以上指标超出适宜区间
  - 标签可点击筛选
- **悬浮卡片**：
  - 实时环境数据（温度、湿度、土壤pH）
  - 生长信息（剩余生长周期、健康度评分）
  - 预警提示（异常指标说明）
- **排序功能**：
  - 默认按种植时间降序
  - 支持按作物名称/生长周期/农场名称排序
- **CRUD操作**：
  - 新增作物（名称、类型、农场、面积、生长周期、适宜环境参数）
  - 编辑作物（除ID外所有字段，支持跨农场修改）
  - 删除作物（权限控制，二次确认）
- **批量操作**：
  - 批量修改种植农场
  - 批量导出Excel（基础信息+7天健康度+环境数据）

#### 4.2 种植区域与生长周期（扩展）
- 作物种植区域管理、生长周期列表与详情页
- 区域维度与环境、农事数据关联展示

### 5. 农资管理模块（库存与采购）

#### 5.1 农资档案
- 农资名称、类型（种子/化肥/农药/工具等）、品牌、规格、单价、计量单位
- 库存数量、安全库存、上下架状态
- 按农场、类型、库存状态筛选；支持增删改与库存调整

#### 5.2 库存预警
- 缺货、库存不足等状态展示
- 与首页库存预警区联动

#### 5.3 农资采购
- 选择农场与已有农资下单；支持在采购流程中**动态新增农资**（名称、类型、单价、规格），创建后自动带入采购单
- 新建采购单默认**自动入库**并更新库存（亦可对待入库记录手动执行入库）
- 采购记录列表：单号、金额、状态（待入库/已入库/已取消）等

#### 5.4 库存流水
- 入库/出库流水查询，与采购、施肥等来源关联

### 6. 农事操作模块（施肥与灌溉）

#### 6.1 操作记录查询
- 汇总**施肥、灌溉**类农事（与采购入库等单据区分）
- 展示农场、**区域、作物**、农资（施肥）、**用水量、灌溉方式、时长**、操作人、时间等
- 支持按农场、区域、作物、类型、时间筛选

#### 6.2 新建施肥 / 灌溉
- 施肥：关联农资出库、用量与单位、方式等
- 灌溉：**用水量、灌溉方式、时长**、区域与作物；与库存、环境建议等后端逻辑配合

### 7. 环境监测模块

#### 7.1 实时数据监测
- 按农场、种植区域展示环境面板（温度、湿度、土壤湿度、光照、pH 等）
- 后台模拟采集定时写入；异常时给出灌溉等提示
- 天气类型、风速、降雨量等（与首页气象等模块配合）

#### 7.2 历史数据查询
- 按时间范围、农场、区域查询历史监测数据
- **历史折线图（ECharts）**：温度、湿度、土壤湿度、光照等多序列对比
- **时间轴策略**：同一天以 **HH:mm** 为主；约 7 日内以 **MM-dd** 为主；约 30 天及以上以 **yyyy-MM** 为主；数据按小时/天/月做桶内**均值聚合**，避免标签拥挤
- **Tooltip** 显示完整时间（yyyy-MM-dd HH:mm:ss）与数值；支持 **dataZoom**（内置缩放 + 底部滑块）拖拽查看区间

#### 7.3 监测报表
- 按日/区间汇总统计（与后端报表接口一致）

### 8. 智能预警模块

#### 8.1 监控设备管理
- 设备新增、编辑、删除；绑定农场与监控覆盖区域
- 展示设备状态（在线/离线/故障）与最近在线时间
- 环境模拟使用系统内置「环境采集终端」设备，与普通设备区分权限

#### 8.2 作物异常记录
- **自动生成**：环境阈值触发、规则扫描等写入异常（来源可区分：手动/规则/环境）
- **手动登记**：异常类型、农场/区域/作物、关联设备、严重程度、处理状态
- 处理状态：未处理 / 已处理 / 已忽略（按角色限制可操作范围）

#### 8.3 异常推送记录
- 异常产生时生成推送记录（如站内信）；支持查看推送历史与已读状态

#### 8.4 处理状态统计
- 异常总数及按状态、类型、农场等维度统计；图表展示；管理员可手动触发规则扫描

#### 8.5 联动逻辑
- 环境数据异常时：写入异常记录与推送；**首页**预警列表可展示并与库存等事件刷新配合

#### 8.6 预警规则与展示（概要）
- 多级预警等级（紧急/普通/提示等）；预警列表、已读标记、滚动排序等

### 9. 首页概览模块

#### 9.1 数据统计与可视化
- 气象、设备、地图、视频等概览（与 `homepage` 接口一致）
- 农场/作物类预警与**库存预警**分区展示

#### 9.2 预警滚动
- 作物/设备类预警消息滚动；未读标记与分页加载
- 与全局库存变更、预警变更等事件配合刷新（具体以当前前端实现为准）

#### 9.3 快捷入口
- 通过主导航进入各业务模块

### 技术选型理由

1. **前端选择React**：
   - 组件化开发，代码复用性强
   - 生态丰富，社区活跃
   - 虚拟DOM提升性能
   - 适合复杂交互场景

2. **后端选择Node.js + Express**：
   - JavaScript全栈开发，技术栈统一
   - 异步I/O，高并发性能好
   - 中间件机制灵活
   - 开发效率高

3. **数据库选择MySQL**：
   - 成熟稳定，广泛应用
   - 支持事务，数据一致性保证
   - 外键约束，数据完整性
   - 索引优化，查询性能好

## 数据库设计

### 数据库概述

- **数据库名**：`smart_agriculture`
- **字符集**：UTF8MB4
- **存储引擎**：InnoDB
- **表总数**：核心基础约 15 张，随业务扩展至 **20 张以上**（含农资采购、库存流水、环境异常日志、预警已读等）

### 核心表结构

#### 1. 用户权限相关表（3张）

**role（角色表）**
- `role_id`：角色编号（主键）
- `role_name`：角色名称

**user（用户表）**
- `user_id`：用户编号（主键）
- `role_id`：角色编号（外键）
- `username`：用户名（唯一）
- `password`：密码（加密）
- `real_name`：真实姓名
- `phone`：手机号
- `farm_id`：所属农场（外键，可空）

**farm_principal（农场负责人绑定表）**
- `binding_id`：绑定编号（主键）
- `farm_id`：农场编号（外键）
- `user_id`：用户编号（外键）
- `principal_type`：负责人类型（主/副）
- `bind_time`：绑定时间
- `bind_by`：绑定操作人（外键）

#### 2. 农场相关表（1张）

**farm（农场表）**
- `farm_id`：农场编号（主键）
- `farm_name`：农场名称
- `address`：地址
- `principal_id`：负责人编号（外键）
- `phone`：联系电话
- `longitude`：经度
- `latitude`：纬度
- `created_at`：创建时间

#### 3. 作物相关表（1张）

**crop（作物表）**
- `crop_id`：作物编号（主键）
- `crop_name`：作物名称
- `farm_id`：农场编号（外键）
- `crop_type`：作物品种
- `crop_category`：作物类型（果蔬/粮食/经济作物）
- `plant_area`：种植区域
- `sow_time`：播种时间
- `growth_cycle`：生长周期（天）
- `suitable_temp_min/max`：适宜温度范围
- `suitable_humidity_min/max`：适宜湿度范围
- `suitable_ph_min/max`：适宜pH范围
- `plant_status`：种植状态（生长中/成熟/已收割）
- `created_at`：创建时间
- `updated_at`：更新时间

#### 4. 环境监测表（1张 + 扩展字段）

**environment_monitor（环境监测表）**
- `monitor_id`：监测编号（主键）
- `farm_id`：农场编号（外键）
- `plant_area`：种植区域（与作物区域对应，便于按区查询）
- `temperature`：温度（℃）
- `humidity`：湿度（%）
- `soil_ph`：土壤pH值
- `soil_moisture`：土壤湿度（%）
- `light_lux`：光照（lux）
- `weather_type`：天气类型（迁移扩展）
- `wind_speed`：风速（m/s）
- `rainfall`：降雨量（mm）
- `monitor_time`：监测时间

**environment_exception_log（环境异常日志表，与作物业务异常表区分）**
- 记录环境规则触发、阈值类说明等，服务于监测与统计，不等同于 `crop_exception` 业务主表

#### 5. 预警相关表（3张）

**monitor_device（监控设备表）**
- `device_id`：设备编号（主键）
- `farm_id`：农场编号（外键）
- `device_name`：设备名称
- `device_category`：设备类别
- `install_location`：安装位置
- `device_status`：设备状态（在线/离线/故障）
- `monitor_area`：监控区域
- `last_online_time`：最后在线时间

**crop_exception（作物异常记录表）**
- `exception_id`：异常编号（主键）
- `crop_id`：作物编号（外键）
- `device_id`：设备编号（外键）
- `exception_type`：异常类型
- `exception_time`：异常时间
- `exception_detail`：异常详情
- `video_url`：视频链接
- `handle_status`：处理状态
- `warning_level`：预警等级
- `scroll_sort`：排序值
- `source_type`：来源（手动/规则/环境等，以实际库字段为准）

**exception_push（异常推送表）**
- `push_id`：推送编号（主键）
- `exception_id`：异常编号（外键）
- `receiver_id`：接收人编号（外键）
- `push_method`：推送方式
- `push_time`：推送时间
- `read_status`：阅读状态

#### 6. 其他业务表（含扩展）

- `agricultural_material`：农资表（含品牌、规格、单价、库存、安全库存、上下架、**计量单位 unit** 等扩展字段）
- `agricultural_material_stock_log`：库存变动流水
- `material_purchase_record`：农资采购单（待入库/已入库/已取消；与入库、流水联动）
- `operation_record`：农事操作记录（扩展 **区域、作物、农资、用量、单位、用水量、灌溉方式、时长** 等）
- `principal_permission`：负责人权限配置表
- `principal_operation_log`：负责人操作日志表
- `warning_read`：预警已读记录表
- `video_device`：视频设备扩展表

### 数据库设计特点

1. **规范化设计**：
   - 符合第三范式（3NF）
   - 消除数据冗余
   - 保证数据一致性

2. **外键约束**：
   - 保证数据完整性
   - 级联删除/更新
   - 防止孤立数据

3. **索引优化**：
   - 主键索引

   - 支持未来功能扩展

## 技术流程

### 1. 用户认证流程

```
用户登录
  ↓
前端发送用户名密码
  ↓
后端验证用户信息
  ↓
生成JWT Token
  ↓
返回Token给前端
  ↓
前端存储Token（localStorage）
  ↓
后续请求携带Token
  ↓
后端验证Token
  ↓
允许/拒绝访问
```

### 2. 数据查询流程

```
前端发起请求
  ↓
Axios拦截器添加Token
  ↓
发送HTTP请求到后端
  ↓
后端中间件验证Token
  ↓
解析请求参数
  ↓
构建SQL查询
  ↓
执行数据库查询
  ↓
处理查询结果
  ↓
返回JSON数据
  ↓
前端接收数据
  ↓
更新UI状态
  ↓
渲染页面
```

### 3. 农场状态计算流程

```
获取农场环境数据
  ↓
查询最新温度/湿度/pH
  ↓
判断指标是否在正常范围
  ↓
计算异常指标数量
  ↓
0个异常 → 正常（绿色）
1个异常 → 预警（黄色）
2+个异常 → 告警（红色）
  ↓
返回状态标签
  ↓
前端显示状态徽章
```

### 4. 作物健康度计算流程

```
获取作物信息
  ↓
查询关联农场环境数据
  ↓
获取作物适宜环境参数
  ↓
计算各项指标偏差
  ↓
温度偏差 + 湿度偏差 + pH偏差
  ↓
计算健康度分数（0-100）
  ↓
健康度 = 100 - (偏差总和 / 最大偏差) * 100
  ↓
返回健康度分数
  ↓
前端显示环形进度条
```

### 5. 权限验证流程

```
用户发起操作请求
  ↓
后端获取用户Token
  ↓
解析Token获取用户信息
  ↓
查询用户角色
  ↓
查询操作权限
  ↓
判断是否有权限
  ↓
有权限 → 执行操作
无权限 → 返回403错误
```

### 6. 数据导出流程

```
用户点击导出按钮
  ↓
前端发送导出请求
  ↓
后端查询数据
  ↓
格式化数据（CSV/Excel）
  ↓
生成文件流
  ↓
设置响应头（Content-Type）
  ↓
返回文件流
  ↓
前端下载文件
```

### 7. 农资采购与入库（概要）

```
前端提交采购单（选定已有农资或先创建农资再采购）
  ↓
后端写入 material_purchase_record（默认待入库或走自动入库）
  ↓
自动入库时：事务内增加库存、写 stock_log、更新采购单为已入库
  ↓
前端刷新列表与库存相关视图
```

## 系统特色

### 1. 智能化特性

- **智能状态判断**：基于环境数据自动计算农场/作物状态
- **健康度分析**：多维度计算作物健康度，提供影响因素
- **智能建议**：基于作物类型、生长周期、环境数据生成灌溉/施肥建议

- **多农场切换**：悬浮面板，快速切换，最近访问记录

- **数据可视化**：折线图、柱状图、饼图展示趋势
- **交互优化**：骨架屏、Toast提示、加载动画

### 3. 权限精细控制

- **角色体系**：超级管理员、农场管理员、普通用户
- **模块权限**：可查看的模块（作物/设备/预警/环境）
- **操作权限**：仅查看/可编辑/可删除
- **多负责人协同**：主/副负责人，权限分级

### 4. 数据安全

- **密码加密**：bcrypt加密存储
- **Token认证**：JWT Token，过期自动失效
- **SQL注入防护**：参数化查询
- **权限验证**：后端中间件验证

### 5. 性能优化

- **数据库索引**：关键字段索引，提升查询速度
- **连接池**：MySQL连接池，复用连接
- **前端优化**：React Hooks优化，减少重复渲染
- **懒加载**：路由懒加载，按需加载组件

## 部署说明

- **Node.js**：v14.0.0 或更高版本
- **MySQL**：v5.7.0 或更高版本
- **npm**：v6.0.0 或更高版本

### 安装步骤

#### 1. 克隆项目

```bash
git clone <repository-url>
cd smart-agriculture-system
```

#### 2. 安装依赖

```bash
# 安装所有依赖（根目录、server、client）
npm run install-all
```

#### 3. 配置数据库

1. 创建MySQL数据库：
```bash
mysql -u root -p
CREATE DATABASE smart_agriculture DEFAULT CHARSET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 执行初始化脚本：
```bash
mysql -u root -p smart_agriculture < server/init.sql
```

3. 配置环境变量（`server/.env`）：
```env
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_password
DB_NAME=smart_agriculture
JWT_SECRET=your_secret_key
PORT=5000
```

#### 4. 启动项目

**开发环境**：
```bash
npm run dev
```

这将同时启动：
- 后端服务器：http://localhost:5000
- 前端开发服务器：http://localhost:3001

**生产环境**：

1. 构建前端：
```bash
cd client
npm run build
```

2. 启动后端：
```bash
cd server
npm start
```

### 默认测试账号

- **管理员**：admin / 123456
- **运维人员**：operator / 123456
- **普通用户**：user / 123456

### 端口配置

- **前端端口**：3001（可在 `client/vite.config.js` 中修改）
- **后端端口**：5000（可在 `server/.env` 中修改）

### 常见问题

1. **数据库连接失败**：
   - 检查MySQL服务是否运行
   - 检查`.env`文件配置是否正确
   - 检查数据库用户权限

2. **端口被占用**：
   - 修改`client/vite.config.js`中的端口
   - 修改`server/.env`中的PORT

3. **依赖安装失败**：
   - 清除缓存：`npm cache clean --force`
   - 删除`node_modules`重新安装

## 数据治理与演示数据

首次部署或需要**补齐展示字段、示例数据、表间关联**时，建议在完成 `init.sql` 与必要迁移后，在 `server` 目录执行：

```bash
cd server
npm run db:full-data
```

该脚本会（可重复执行，尽量幂等）：

- 扩展 `environment_monitor` 等与路由一致的字段，并补近期监测数据，避免历史曲线无数据
- 调用农资模块 schema 保障，为农资补 **品牌、规格、计量单位** 等缺省值
- 补全作物 **crop_name**、默认类别与状态等
- 扩展并回填 `operation_record`（灌溉的用水量、方式、时长、区域与作物关联等）
- 按农场与种植区域补充 **monitor_device** 监测点
- 在农事记录偏少时插入少量演示灌溉/施肥（带固定备注，避免重复堆积）

另可选用：

```bash
node server/seed-demo-linkage.js
```

用于库存预警演示（缺货/库存不足）及简单灌溉演示行。

**说明**：`environment_exception_log` 与 `crop_exception` 职责不同，不建议合并为单表；业务预警以 `crop_exception` 为主链。

## 总结

智慧农业管理系统是一个基于 **React + Vite + Node.js（Express）+ MySQL** 的现代化农业管理平台，前后端分离。当前实现涵盖**农场与作物管理、农资与采购入库、库存与流水、农事操作（施肥/灌溉）、环境监测（实时/历史图表）、智能预警（设备—异常—推送—统计—首页联动）、首页与仪表盘**等能力，并支持通过数据治理脚本快速得到可用于演示的完整关联数据。

### 技术亮点

- ✅ 前后端分离，接口 RESTful，JWT 鉴权
- ✅ 农资采购支持动态建档与默认自动入库，减少流程断点
- ✅ 农事与环境、库存流水可追溯
- ✅ 智能预警全链路 + 首页/库存事件刷新思路
- ✅ 历史环境图表按时间范围自适应轴粒度与聚合，ECharts + dataZoom 提升可读性
- ✅ 数据库外键与迁移扩展并存，可按脚本补全演示数据

### 未来扩展

- 移动端或小程序
- 更多物联网设备协议接入
- 离线分析与大数据看板
- 更细粒度 AI 推荐与病虫害模型
